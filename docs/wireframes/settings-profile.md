# Profile Settings Page - Wireframe

## Purpose

Allow all users (admins and employees) to manage their profile information, preferences, and security settings.

## User Story

**As a user, I want to:**
- Update my display name
- View my email, role, and team (read-only)
- Choose my preferred theme (Light/Dark/System)
- Control notification preferences
- Change my password securely

## Component Specifications

### Page Header

**Breadcrumb:** Settings > Profile
**Title:** Profile & Preferences
**Subtitle:** Manage your personal settings and preferences

**Styling:**
- Title: `text-3xl font-bold`
- Subtitle: `text-muted-foreground text-base`
- Spacing: `space-y-1 mb-8`

### Profile Information Card

**Type:** Form section with Card component

**Fields:**

1. **Display Name** (Editable, Required)
   - Type: Text input
   - Max length: 255 characters
   - Validation: Required, min 2 chars
   - Helper: "Your name as it appears throughout the application"

2. **Email** (Read-only)
   - Type: Plain text
   - Typography: `text-sm text-muted-foreground`

3. **Role** (Read-only)
   - Type: Badge component
   - Variants:
     - Admin: `variant="default"`
     - Manager: `variant="secondary"`
     - Employee: `variant="outline"`

4. **Team** (Read-only)
   - Type: Badge component
   - Variant: `variant="secondary"`
   - Displays: Team name or "No Team" if null

**Layout:**
- Card with padding: `p-6`
- Display name field: Full width
- Email: Full width, smaller text
- Role + Team: Two-column grid (`grid grid-cols-2 gap-4`)

### Appearance Card

**Type:** Form section with Card component

**Fields:**

1. **Theme** (Select dropdown)
   - Options:
     - "Light"
     - "Dark"
     - "System Preference"
   - Default: "System Preference"
   - Helper: "Choose your preferred color theme"

**Layout:**
- Card with padding: `p-6`
- Select: Full width

### Notifications Card

**Type:** Form section with Card component

**Fields:**

All fields are Switch components (toggle on/off)

1. **Email Notifications**
   - Label: "Email Notifications"
   - Description: "Receive notifications via email"
   - Default: true

2. **In-App Notifications**
   - Label: "In-App Notifications"
   - Description: "Receive notifications in the application"
   - Default: true

3. **Agent Activity Alerts**
   - Label: "Agent Activity Alerts"
   - Description: "Get notified when agents complete tasks"
   - Default: false

4. **Weekly Summary**
   - Label: "Weekly Summary"
   - Description: "Receive weekly usage summary email"
   - Default: true

**Layout:**
- Card with padding: `p-6`
- Each item: `flex justify-between items-start space-y-4`
- Label + description on left, switch on right

### Security Card

**Type:** Separate form section (not tied to main form state)

**Fields:**

1. **Current Password** (Required)
   - Type: Password input
   - Show/hide toggle button
   - Validation: Required

2. **New Password** (Required)
   - Type: Password input
   - Show/hide toggle button
   - Validation: Min 8 chars, uppercase, lowercase, number
   - Real-time strength indicator

3. **Confirm New Password** (Required)
   - Type: Password input
   - Show/hide toggle button
   - Validation: Must match new password

**Update Password Button:**
- Variant: `variant="default"`
- Type: `submit` (for password form)
- Enabled: Only when all password fields filled and valid
- Loading state: Spinner + "Updating..."

**Layout:**
- Card with padding: `p-6`
- Fields spacing: `space-y-4`
- Button: Left-aligned, below fields

### Action Buttons

**Note:** These buttons control Profile, Appearance, and Notifications sections only. Password has its own button.

**Cancel Button:**
- Variant: `variant="outline"`
- Enabled: Only when form is dirty
- Action: Reset form to original values

**Save Changes Button:**
- Variant: `variant="default"`
- Enabled: Only when form is dirty AND valid
- Action: Submit profile/preferences form
- Loading state: Spinner + "Saving..."

**Layout:**
- Container: `flex gap-3 mt-8`
- Alignment: Left-aligned

**Calculation:**
- Base score: 0
- +20 points: 8+ characters
- +20 points: 12+ characters
- +20 points: Contains uppercase
- +20 points: Contains lowercase

- +20 points: Contains special char

- 0-40: Weak (red)
- 41-60: Fair (orange)
- 61-80: Good (yellow)
- 81-100: Strong (green)
- 100+: Excellent (green)

```typescript
const passwordSchema = z.object({
```

```typescript
function handleThemeChange(theme: 'light' | 'dark' | 'system') {
  // 1. Immediately update UI (optimistic)
  setTheme(theme)

  // 2. Mark form as dirty
  profileForm.setValue('theme', theme, { shouldDirty: true })

  // 3. Save will happen when user clicks "Save Changes"
}
```
