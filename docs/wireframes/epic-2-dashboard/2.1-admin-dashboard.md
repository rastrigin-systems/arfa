## Component Specifications

### 1. Page Header
- **Page Title:** "Dashboard" - 32px, bold
- **Welcome Message:** "Welcome back, [Name]!" - 16px, muted text
- **Spacing:** 8px between title and welcome message

### 2. Layout Grid
- **Desktop (1024px+):** 2-column layout for main sections
  - Left column: Pending Approvals (40% width)
  - Right column: Activity Timeline (60% width)
  - Below: Quick Stats (full width, 4 columns)
  - Below: Quick Actions (full width, 4 buttons)

- **Tablet (768px-1023px):** Stack sections vertically
  - Pending Approvals (full width)
  - Activity Timeline (full width)
  - Quick Stats (2x2 grid)
  - Quick Actions (2x2 grid)

- **Mobile (320px-767px):** Single column
  - All sections full width
  - Quick Stats: 2x2 grid
  - Quick Actions: Vertical stack

## User Flows

### Flow 1: Admin Logs In
```
1. Admin navigates to /login
2. Enters credentials
3. Clicks "Login"
4. API authenticates
5. Redirect to /dashboard
6. Dashboard loads (shows loading skeletons)
7. API calls execute in parallel:
   - GET /api/v1/approvals/pending?limit=5
   - GET /api/v1/activity-logs?limit=10
   - GET /api/v1/organizations/current/stats
8. Data populates sections as responses arrive
9. Dashboard fully loaded
```

### Flow 2: View All Pending Requests
```
1. Admin on /dashboard
2. Sees "Pending Approvals (3)" section
3. Clicks "View All Requests" button
4. Navigate to /approvals
5. Full list of pending requests shown
```

### Flow 3: View All Activity
```
1. Admin on /dashboard
2. Sees "Activity Timeline" section
3. Clicks "View All Activity" button
4. Navigate to /activity
5. Complete activity log shown
```

### Flow 4: Quick Action - Manage Employees
```
1. Admin on /dashboard
2. Sees "Quick Actions" section
3. Clicks "Manage Employees" button
4. Navigate to /employees
5. Employee management page loads
```

### Flow 5: Error Handling - Pending Approvals Fails
```
1. Dashboard loads
2. API call to /api/v1/approvals/pending fails
3. Error state shown in Pending Approvals section
4. Other sections continue to load successfully
5. Admin clicks "Retry" button
6. API call retries
7. Success: Section loads with data
```

## Technical Implementation Notes

**Current State:**
- Basic shell exists with welcome message and placeholder cards
- Needs rework to implement sections as specified

**Implementation Approach:**
1. Create reusable components:
   - `PendingApprovalsList`
   - `ActivityTimelineList`
   - `QuickStatCard`
   - `QuickActionButton`

2. Data fetching:
   - Use React Server Components for initial load
   - Or: Client-side fetching with SWR/React Query
   - Optional: Auto-refresh every 60 seconds (polling)

3. Error handling:
   - Each section handles its own errors independently
   - Failed sections don't block others
   - Retry mechanism for each section

4. Loading states:
   - Skeleton loaders while data fetching
   - Smooth transitions when data arrives

**Performance:**
- Parallel API calls for faster load
- Limit data to most recent (5 approvals, 10 activities)
- Cache stats with 60-second TTL

## Design Decisions

1. **Admin-only dashboard** - Not for regular employees, focused on management tasks
2. **Priority layout** - Pending approvals at top (most important)
3. **Activity timeline** - Provides context and visibility
4. **Quick stats** - At-a-glance metrics for decision-making
5. **Quick actions** - Fast navigation to common tasks
6. **Empty states** - Encouraging messages, not just "No data"
7. **Error resilience** - Each section independent, retry mechanism
8. **Auto-refresh optional** - Real-time updates for approvals
9. **Responsive design** - Works on all screen sizes
10. **Accessible** - Keyboard, screen reader, WCAG AA compliant
