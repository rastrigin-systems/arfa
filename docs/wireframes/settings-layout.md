## Component Specifications

### Settings Sidebar (Nested)

**Location:** Rendered within settings pages content area (not main sidebar)

**Width:**
- Desktop: 240px fixed
- Tablet: Hidden (tabs instead)
- Mobile: Horizontal tabs (full width)

**Items:**

| Icon | Label | Route | Visibility | State |
|------|-------|-------|------------|-------|
| Building2 | Organization | /settings/organization | Admin only | Active |
| User | Profile | /settings/profile | All users | Active |
| CreditCard | Billing | /settings/billing | All users | Disabled (Coming Soon) |
| Shield | Security | /settings/security | All users | Disabled (Coming Soon) |
| Puzzle | Integrations | /settings/integrations | All users | Disabled (Coming Soon) |

**States:**

1. **Active (Current Page):**
   - Background: `bg-primary`
   - Text: `text-primary-foreground`
   - Border: `border-l-4 border-primary`
   - Chevron: Visible `>`

2. **Inactive (Clickable):**
   - Background: `transparent`
   - Text: `text-foreground`
   - Hover: `hover:bg-muted`

3. **Disabled (Coming Soon):**
   - Background: `transparent`
   - Text: `text-muted-foreground`
   - Cursor: `not-allowed`
   - Badge: "Coming Soon" (gray)
   - No hover effect

### Breadcrumb

**Format:** `Settings > [Current Section]`

**Examples:**
- `Settings > Organization`
- `Settings > Profile`

**Location:** Top of content area, below header

### Route Structure

```
/settings â†’ redirects based on role:
  - Admin â†’ /settings/organization
  - Employee â†’ /settings/profile

/settings/organization â†’ Organization page (admin only)
/settings/profile â†’ Profile page (all users)
/settings/billing â†’ 404 or "Coming Soon" page
/settings/security â†’ 404 or "Coming Soon" page
/settings/integrations â†’ 404 or "Coming Soon" page
```

### Permission Logic

```typescript
// Pseudocode
if (user.role.permissions.includes('manage_organization')) {
  showOrganizationLink = true
  defaultRoute = '/settings/organization'
} else {
  showOrganizationLink = false
  defaultRoute = '/settings/profile'
}
```

## Interaction Flows

### 1. Admin Opens Settings

```
1. Click "Settings" in main sidebar
2. Navigate to /settings
3. Server checks role â†’ redirect to /settings/organization
4. Page loads with Organization sidebar item active
5. Organization settings content displayed
```

### 2. Employee Opens Settings

```
1. Click "Settings" in main sidebar
2. Navigate to /settings
3. Server checks role â†’ redirect to /settings/profile
4. Page loads with Profile sidebar item active
5. Profile settings content displayed
6. Organization link NOT visible in sidebar
```

### 3. Navigate Between Sections

```
1. Click "Profile" in settings sidebar
2. Client-side navigation to /settings/profile
3. Sidebar updates active state
4. Content area re-renders with profile form
5. URL updates in browser
```

### 4. Try to Access Disabled Section

```
1. Click "Billing" (Coming Soon)
2. No action (cursor: not-allowed)
3. Optionally: Show toast "Coming soon"
```

- Active item: `bg-primary text-primary-foreground`

## Accessibility Notes

- Settings sidebar must be navigable via keyboard
- Active item announced by screen reader: "Organization settings, current page"
- Disabled items announced: "Billing settings, coming soon, button disabled"
- Breadcrumb is a `<nav>` with `aria-label="Breadcrumb"`
- Content area has `<main>` role
- Sidebar has `<nav>` role with `aria-label="Settings navigation"`
