## User Flows

### Empty State Flow (No Configs)
1. Page loads
2. API returns 0 configurations
3. Show empty state with gear icon
4. Display "No agent configurations yet" message
5. Show "Create Configuration" CTA
6. Optional "Learn more" link
7. Click CTA â†’ Open create modal

### Empty State Flow (Filtered)
1. User applies filters or searches
2. API returns 0 results
3. Show empty state with search icon
4. Display "No configurations match" message
5. Show "Clear All Filters" button
6. Click Clear â†’ Reset filters to defaults
7. Table refreshes with all configs

### Loading State Flow
1. Page loads or filters change
2. Show skeleton rows (5 rows)
3. "Loading configurations..." text below table
4. API responds
5. Skeleton replaced with data OR empty state OR error state

### Error State Flow
1. API call fails (network, timeout, 500 error)
2. Show inline error banner at top
3. Show error empty state with warning icon
4. Display error message and suggestion
5. Provide "Try Again" button
6. Click Try Again â†’ Retry API call
7. Success â†’ Show data OR repeat error flow

## Edge Cases

### Network Timeout
- Show error state after 30 seconds
- Error message: "Request timed out. Check your connection and try again."

### Partial Data Load
- If some configs load but API returns errors: Show loaded data + error banner at top
- Banner: "Some configurations could not be loaded. [Retry]"

### Concurrent Filters
- Multiple filters active + no results: Show count of active filters in empty state
- Example: "No configurations match your 3 active filters"

### Permission Denied
- If user lacks permission to view configs: Show error state
- Message: "You don't have permission to view configurations. Contact your administrator."
- No "Try Again" button (won't resolve)
